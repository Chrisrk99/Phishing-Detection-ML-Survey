The Role of High-Performance Computing in Enhancing Machine Learning for Phishing Detection

Phishing attacks remain a formidable cybersecurity threat, relentlessly evolving to outpace traditional detection methods. Machine learning (ML) has emerged as a critical countermeasure, enabling proactive and adaptive identification of phishing attempts. Yet, the computational demands of MLâ€”particularly for processing vast datasets or achieving real-time detectionâ€”often constrain its effectiveness. High-performance computing (HPC) addresses this bottleneck, delivering the power needed to boost the speed, scalability, and efficiency of ML models. This survey investigates the integration of HPC with ML to tackle phishing detection challenges, focusing on three key areas: ML Implementations on HPC Architectures, Advanced HPC Tools and Technologies, and Impact on Performance Metrics.

  • I. Fette, N. Sadeh, and A. Tomasic, "Learning to Detect Phishing Emails," Proc. 16th Int. Conf. World Wide Web (WWW '07), 2007. PDF

    This paper introduces PILFER, a machine learning-based email filter using features specifically designed to detect phishing. It combines structural and content-based features, achieving over 96% accuracy with low false positives. The study emphasizes early-stage email-level defenses over browser-based detection.

  • M. Khonji, Y. Iraqi, and A. Jones, "Phishing Detection: A Literature Survey," IEEE Commun. Surveys Tuts., 2013. PDF

    This survey categorizes phishing solutions across detection, prevention, and mitigation. It evaluates blacklists, heuristic and machine learning-based detection, and human factors. The authors emphasize the importance of layered defenses and highlight gaps in current NLP and user-awareness solutions.

  • S. Abu-Nimeh, D. Nappa, X. Wang, and S. Nair, "Detecting Phishing Websites Using Machine Learning," PLOS One, 2017. PDF

    This study develops a deep learning model using LSTM-based RNNs to detect phishing URLs. It achieves high accuracy using only URL features and demonstrates strong generalization against domain generation attacks. The method provides lightweight, real-time phishing detection.

  • R. Dhamija and J. D. Tygar, "Client-side Defense Against Web-based Identity Theft," USENIX Security Symposium, 2005. PDF

    Through user studies, this paper reveals how easily users can be tricked by visually deceptive websites, even with security indicators present. It highlights the limitations of technical indicators and advocates for more effective user interface design to reduce phishing risk.

  • S. Garera et al., "A Framework for Detecting Phishing Websites," IEEE Symposium on Security and Privacy (S&P), 2007. PDF

    CANTINA+ is a feature-rich anti-phishing framework combining webpage content, login form analysis, and similarity hashing. It introduces eight novel features and two filtering stages to reduce false positives. The system achieves over 92% true positives and very low false detection rates.
